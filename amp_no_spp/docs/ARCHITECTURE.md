# ESP32 Audio Controller - Документація архітектури

## Огляд

Аудіоконтролер на базі ESP32 з наступними можливостями:
- **Bluetooth A2DP** - потокове аудіо з телефону
- **Bluetooth SPP** - керування з Android додатку
- **TDA7318** - аудіопроцесор (гучність, НЧ, ВЧ, баланс, підсилення, перемикання входів)
- **UDA1334A** - I2S ЦАП для виводу аудіо
- **ST7789** - TFT дисплей 170x320
- **IR RC-5** - керування пультом
- **Енкодер** - регулювання параметрів
- **6 кнопок** - перемикання входів та керування відтворенням
- **EEPROM** - збереження налаштувань

## Апаратна архітектура

```
┌─────────────────────────────────────────────────────────────┐
│                         ESP32-WROOM-32                       │
│                                                              │
│  ┌─────────────┐                                            │
│  │  Bluetooth   │←─── Android телефон (A2DP + SPP)          │
│  │  (WiFi/BT)   │                                            │
│  └──────┬──────┘                                            │
│         │ I2S аудіо                                          │
│         ├──────────────────→ UDA1334A (ЦАП) → Підсилювач    │
│         │ I2C керування                                      │
│         ├──────────────────→ TDA7318 (аудіопроцесор)        │
│         │ SPI                                                 │
│         ├──────────────────→ ST7789 (170x320 TFT)           │
│  ┌──────┴──────┐                                            │
│  │   GPIOs     │                                            │
│  │             │                                            │
│  │  - Енкодер  │ (CLK, DT, SW)                             │
│  │  - Кнопки   │ (PWR, UP, DOWN, LEFT, OK, RIGHT)          │
│  │  - IR RC-5  │ (GPIO15)                                  │
│  └─────────────┘                                            │
└─────────────────────────────────────────────────────────────┘
```

### Карта пінів

| Периферія | Сигнал | GPIO | Примітки |
|------------|--------|------|----------|
| **UDA1334A (I2S)** | BCK | 26 | Тактовий сигнал |
| | WS/LCK | 25 | Вибір слова |
| | DIN | 27 | Дані з ESP32 |
| **TDA7318 (I2C)** | SDA | 21 | Дані |
| | SCL | 22 | Тактування |
| **ST7789 (SPI)** | MOSI | 23 | Master out |
| | SCLK | 18 | Тактування |
| | CS | 5 | Вибір чіпу |
| | DC | 16 | Дані/команда |
| | RST | 17 | Скидання |
| | BL | 4 | Підсвітка (PWM) |
| **Енкодер** | CLK | 13 | Квадратура A |
| | DT | 14 | Квадратура B |
| | SW | 19 | Кнопка |
| **IR приймач** | DATA | 15 | Протокол RC-5 |
| **Кнопки** | PWR | 33 | Живлення/очікування |
| | UP | 32 | Вхід вверх |
| | DOWN | 36 | Вхід вниз |
| | LEFT | 35 | Попередній трек |
| | OK | 34 | Play/pause |
| | RIGHT | 39 | Наступний трек |

**Примітка:** Всі кнопки та піни енкодера підтягнуті через 12кОм до +3.3В

### Опис завдань

#### 1. Завдання Bluetooth аудіо (Пріоритет 24)

**Призначення:** Обробка потокового аудіо A2DP з телефону

**Відповідальність:**
- Ініціалізація A2DP Sink
- Прийом аудіопотоку
- Вивід на I2S (UDA1334A)
- Обробка команд AVRCP (play/pause, next, prev)
- Моніторинг стану підключення

**Розмір стеку:** 4096 байт

**Взаємодія з чергами:**
- Приймає: Немає (найвищий пріоритет, прямі callback)
- Відправляє: Немає

---

#### 2. Завдання Bluetooth керування (Пріоритет 20)

**Призначення:** Обробка комунікації SPP з Android додатком

**Відповідальність:**
- Ініціалізація SPP серверу
- Прийом підключень від Android додатку
- Парсинг JSON команд
- Відправка оновлень статусу (тільки при зміні)
- Команди черги TDA7318

**Розмір стеку:** 3072 байт

**Взаємодія з чергами:**
- Приймає: `bt_control_queue` (команди від Android)
- Відправляє: `tda7318_queue` (зміни параметрів)
- Відправляє: JSON статус назад на Android через SPP

**Протокол:**
```json
// При підключенні - відправити повний статус
{"type": "status", "input": 0, "volume": 40, "bass": 0, ...}

// Команди від Android
{"cmd": "set_volume", "value": 40}
{"cmd": "set_input", "value": 1}

// Оновлення статусу (тільки при зміні)
{"type": "update", "param": "volume", "value": 40}
```

---

#### 3. Завдання TDA7318 (Пріоритет 16)

**Призначення:** Керування аудіопроцесором TDA7318

**Відповідальність:**
- Ініціалізація I2C комунікації
- Виконання команд з черги
- Застосування налаштувань до регістрів TDA7318
- Debounce швидких змін
- Кешування поточних налаштувань

**Розмір стеку:** 2048 байт

**Взаємодія з чергами:**
- Приймає: `tda7318_queue` (команди оновлення параметрів)
- Відправляє: Немає

**Карта регістрів TDA7318:**
| Регістр | Функція | Діапазон | За замовчуванням |
|----------|----------|-------|---------|
| 0x00 | Гучність | 0-63 (-78.75 до 0 дБ) | 40 |
| 0x01 | Підсилення | 0-3 (0 до +13.5 дБ) | 0 |
| 0x02 | НЧ | -7 до +7 (-10.5 до +10.5 дБ) | 0 |
| 0x03 | Середні | -7 до +7 | 0 |
| 0x04 | ВЧ | -7 до +7 (-10.5 до +10.5 дБ) | 0 |
| 0x05 | Баланс | -31 до +31 | 0 |
| 0x06 | Лівий аттенюатор | 0-15 | 0 |
| 0x07 | Правий аттенюатор | 0-15 | 0 |
| 0x08 | Вибір входу | 0-3 | 0 |
| 0x09 | Mute | 0-1 | 0 |

#### 4. Завдання дисплея (Пріоритет 12)

**Призначення:** Візуалізація інтерфейсу на дисплеї ST7789

**Відповідальність:**
- Ініціалізація SPI та дисплея
- Малювання елементів UI (шкала гучності, назва входу, тощо)
- Оновлення дисплея по подіях з черги
- Обробка анімацій/переходів
- Керування підсвіткою

**Розмір стеку:** 4096 байт (великий для framebuffer)

**Взаємодія з чергами:**
- Приймає: `display_queue` (події оновлення)
- Відправляє: Немає

#### 5. Завдання вводу (Пріоритет 8)

**Призначення:** Зчитування всіх пристроїв вводу

**Відповідальність:**
- Зчитування енкодера (обертання + кнопка)
- Зчитування 6 кнопок
- Декодування сигналів IR RC-5
- Debounce ввідних сигналів
- Відправка подій у чергу

**Розмір стеку:** 2048 байт

**Взаємодія з чергами:**
- Приймає: Немає (зчитує GPIO напряму)
- Відправляє: `input_queue` (події для інших завдань)

**Функції кнопок:**
| Кнопка | Коротке натискання | Довге натискання |
|--------|-------------------|------------------|
| PWR | Перемкнути очікування | - |
| UP | Вхід +1 | - |
| DOWN | Вхід -1 | - |
| LEFT | Попередній трек | - |
| OK | Play/Pause | - |
| RIGHT | Наступний трек | - |
| Енкодер SW | Цикл параметрів | - |

**Цикл параметрів енкодера:**
Гучність → НЧ → ВЧ → Баланс → Підсилення → (назад) Гучність

**Команди IR RC-5:**
| Код | Функція |
|------|----------|
| 0x0C | Живлення |
| 0x10 | Вверх |
| 0x11 | Вниз |
| 0x12 | Вліво |
| 0x13 | OK |
| 0x14 | Вправо |
| 0x15 | Гучність + |
| 0x16 | Гучність - |
| 0x0D | Mute |

#### 6. Завдання системи (Пріоритет 4)

**Призначення:** Ініціалізація системи та моніторинг

**Відповідальність:**

- Моніторинг пам'яті heap

**Розмір стеку:** 2048 байт

## Потік даних

### Регулювання гучності енкодером

```
[Енкодер] 
    │ (зчитування GPIO)
    ▼
[Завдання вводу]
    │ (черга: EVENT_ENCODER_ROTATE, значення=+1)
    ▼
[Логіка додатку]
    │ (оновлення кешованої гучності)
    ▼
[Черга: tda7318_queue] ────► [Завдання TDA7318] ────► I2C ────► [TDA7318]
    │
    ▼
[Черга: display_queue] ────► [Завдання дисплея] ────► SPI ────► [ST7789]
    │
    ▼
[SPP відправка] ───────────► [Завдання BT керування] ────► [Android додаток]
```

### Перемикання входу Bluetooth (від Android додатку)

```
[Android додаток]
    │ (SPP: {"cmd": "set_input", "value": 1})
    ▼
[Завдання BT керування]
    │ (парсинг JSON)
    ▼
[Черга: tda7318_queue] ────► [Завдання TDA7318] ────► I2C ────► [TDA7318]
    │
    ├─► [Черга: display_queue] ────► [Завдання дисплея] ────► [ST7789]
    │
    └─► [Збереження EEPROM] ────► Flash (збереження налаштувань)
```

### Кнопка живлення → Режим очікування

```
[Кнопка PWR]
    │ (зчитування GPIO)
    ▼
[Завдання вводу]
    │ (черга: EVENT_BUTTON_PRESS, значення=BTN_FUNC_PWR)
    ▼
[Системна логіка]
    │ (перемкнути прапорець очікування)
    ├─► Mute TDA7318
    ├─► Дисплей вимкнено (підсвітка = 0)
    ├─► Збереження в EEPROM
    └─► Сповіщення Android додатку через SPP
```

## Карта пам'яті EEPROM

```
Адреса 0x0000:
  [Магія: 0xA5] [Поточний вхід: 0-3] [Очікування: 0/1] [Резерв]

Адреса 0x0010: Вхід 0 (Bluetooth) налаштування
  [Гучність] [НЧ] [ВЧ] [Баланс] [Підсилення] [Ост. параметр] [Рез] [Рез]

Адреса 0x0020: Вхід 1 (Computer) налаштування
  [Гучність] [НЧ] [ВЧ] [Баланс] [Підсилення] [Ост. параметр] [Рез] [Рез]

Адреса 0x0030: Вхід 2 (TV Box) налаштування
  [Гучність] [НЧ] [ВЧ] [Баланс] [Підсилення] [Ост. параметр] [Рез] [Рез]

Адреса 0x0040: Вхід 3 (AUX) налаштування
  [Гучність] [НЧ] [ВЧ] [Баланс] [Підсилення] [Ост. параметр] [Рез] [Рез]
```

**Всього: 80 байт використано з 512 доступних**

## Протокол Bluetooth SPP

### Потік підключення

```
1. Android додаток підключається до SPP серверу ESP32
2. ESP32 відправляє повний статус одразу:
   {
     "type": "status",
     "input": 0,
     "volume": 40,
     "bass": 0,
     "treble": 0,
     "balance": 0,
     "gain": 0,
     "mute": false,
     "standby": false
   }

3. Android додаток відправляє команди
4. ESP32 виконує команду, застосовує до TDA7318
5. ESP32 відправляє оновлення ТІЛЬКИ якщо значення змінилося:
   {"type": "update", "param": "volume", "value": 40}
```

### Довідник команд

| Команда | Опис | Діапазон значень |
|---------|-------------|-------------|
| `set_volume` | Встановити гучність | 0-63 |
| `set_bass` | Встановити НЧ | -7 до +7 |
| `set_treble` | Встановити ВЧ | -7 до +7 |
| `set_balance` | Встановити баланс | -31 до +31 |
| `set_gain` | Встановити підсилення | 0-3 |
| `set_input` | Перемкнути вхід | 0-3 |
| `mute` | Перемкнути mute | true/false |
| `standby` | Перемкнути очікування | true/false |
| `get_status` | Запит повного статусу | - |

### Типи оновлень статусу

| Тип | Коли відправляється |
|------|-------------------|
| `status` | При початковому підключенні |
| `update` | При зміні будь-якого параметру |
| `input_changed` | При перемиканні джерела входу |
| `standby_changed` | При зміні стану очікування |
| `bt_connected` | При підключенні/відключенні A2DP |

## Обробка помилок

### Помилки комунікації I2C
- Повторна спроба 3 рази з затримкою 10мс
- Запис помилки в Serial (якщо увімкнено DEBUG_SERIAL)
- Продовжити роботу з кешованими значеннями

### Відключення Bluetooth
- A2DP відключено: аудіо зупиняється, дисплей показує "Відключено"
- SPP відключено: жодних дій, чекати перепідключення

### Пошкодження EEPROM
- Перевірка магічного байту (0xA5) при старті
- Якщо недійсний - ініціалізація значеннями за замовчуванням
- Збереження значень за замовчуванням в EEPROM

### Watchdog таймер
- Кожне завдання має завершитись протягом 1 секунди
- Довгі операції повинні використовувати vTaskDelay()

## Керування живленням

### Активний режим
- Всі завдання працюють
- Дисплей увімкнено
- Bluetooth реклама
- WiFi вимкнено (економія ~50мА)

### Режим очікування
- TDA7318 muted
- Підсвітка дисплея вимкнена
- Завдання вводу опитує (низька частота)
- Bluetooth все ще активний (для дистанційного пробудження)
- Струм: ~10-20мА

### Джерела пробудження
- Будь-яке натискання кнопки
- Обертання енкодера
- Сигнал IR пульта
- Подія підключення Bluetooth
